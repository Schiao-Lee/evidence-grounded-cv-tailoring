# 从这里开始 · 新手中文上手指南

> 🌐 [English](START_HERE_en.md) | 中文

> 这份指南给第一次用的人。你**不需要懂代码**——你只要会跟 AI 说人话,并守住一条底线。
> (其余文档是英文的;真正操作时把下面的提示词直接复制粘贴即可。)

---

## 先记住这一件事(最重要)

这**不是**一个"帮我把简历写好看"的按钮。它的全部价值是:**每一句都真、都查得到。**

岗位要求里有你没有的东西,它会如实记成一条 *gap*(差距),**绝不替你编**。
你越老实,它帮你越多。请记住:

> **它要"往大里说"的时候,你负责说"不"。剩下的交给它。**

---

## 第 0 步:装好环境(唯一需要动手装的一步)

这一步对非技术的人最容易卡。需要在你电脑上装好:

- **Claude Code**(跟 AI 对话的工具,需要一个 Claude 账号)
- **XeLaTeX**(把简历排成 PDF)
- **poppler**(检查 PDF)

具体安装命令见英文 `README.md` 的 *Prerequisites* 表(macOS 一行行复制即可)。

> 💡 建议:这一步请让**帮你的人陪你装一次**(当面或屏幕共享)。装完打开 kit 文件夹,
> 后面就全靠对话了,你自己能走。

装好后:打开 Claude Code,确认它打开的是 `cv-workflow-kit` 这个文件夹。然后开始下面四句。

---

## 开场四句(按顺序,逐条复制粘贴)

### ① 先认识它 + 自检环境(先别动手)

```
请先读这个文件夹里的 README.md 和 CLAUDE.md,然后用大白话告诉我三件事:
1) 这个工具到底是干嘛的;
2) 我这台电脑还差什么才能正常用它(检查 XeLaTeX、poppler 装好没);
3) 我需要提供哪些关于我自己的信息。
先不要做任何修改。
```

### ② 用"假人"空跑一遍(建立信心 + 验证装好了)

```
用 examples/toy_profile 当作 profile、examples/toy_jds/data_analyst.txt 当 JD,
完整跑一遍 prompts/tailor_per_job.md 的流程,最后把 QA gate 的结果表给我看。
这是假人测试,不要碰我的真实信息。
```

看到一份假人的简历被生成出来、质检全部 PASS——你就明白整条流程长什么样了。

### ③ 建立你自己的资料(核心一步)

```
现在帮我建立我自己的资料。请按 BOOTSTRAP.md 来,一步一步采访我,从 Step A 开始。
严格遵守 CLAUDE.md 的诚实原则:只记录真实、有依据的事实。
我如果含糊或夸大,请追问我、或者拦住我。
```

AI 会像访谈一样,一项项问你(姓名、联系方式、学历、语言、做过什么项目……)。
你只要**如实回答**,它会帮你把资料(profile + 你的"总简历")都建好。

### ④ 投第一个真实岗位

```
我要投一个真实岗位,按 prompts/tailor_per_job.md 来。
公司、职位、JD 在这里:【把招聘信息粘进来】
请做到 qa_one_job.py 全部 PASS 为止,中间需要我拍板的地方问我。
```

---

## 怎么跟它对话(3 条就够)

1. **它问你,你就如实答。** 不确定的就说"不确定",别猜着填。
2. **它要夸大时,拦住它。** 比如它把"课程项目"写成"上线系统",你就说:"这只是课程作业,改回去。"
3. **以质检为准。** 只有看到 `qa_one_job.py` **全部 PASS**,这份简历才算完成;没过就让它继续修。

---

## 以后怎么用(日常)

设置只做一次(第 ①②③ 句)。之后每投一个岗位,**只用第 ④ 句**:换公司、换 JD,跑到质检全过即可。

每份简历跑完,它还会在 `PHONE_INTERVIEW_PREP/` 里给你生成一份电话面试小抄
(含怎么诚实回答 gap),接到突然来电也不会乱讲。

> 🚀 **进阶(熟练之后再看):** 想一次投很多岗?把一堆 LinkedIn JD 粘成一个文件,
> 用批量模式一行一份地跑——见英文 `prompts/batch_driver.md`。一开始不用管它,
> 先把单个岗位跑顺。

## 这个工具箱里有什么(知道大概就行)

你基本只会碰到 `profile.yaml` 和你的简历;其余都是 AI 在用。

| 你会碰到的 | 是什么 |
|------------|--------|
| `profile.yaml` | **你的资料卡**:姓名、联系方式、学历、语言、以及"我绝不夸大的事实"清单。设置时建一次,以后偶尔更新 |
| `MASTER_CV/` 里你的"总简历" | 你**所有**经历的全集。每份投递都是从它**删减**出来的,不会往里加新东西 |
| `JOB_APPLICATIONS/<公司>/` | 每投一个岗,这里多一个文件夹,放裁好的简历 PDF + 报告 |
| `PHONE_INTERVIEW_PREP/` | 每份简历配一张电话面试小抄 |

| AI 在用的(你不用管) | 是什么 |
|------------------------|--------|
| `CLAUDE.md` | 给 AI 立的规矩(诚实优先、必须过质检) |
| `prompts/tailor_per_job.md` | 裁剪一个岗位的完整流程 |
| `BOOTSTRAP.md` | 第一次帮你建资料的访谈脚本 |
| `tools/` | 质检脚本(`qa_one_job.py` 等) |
| `examples/` | 假人样例,练手用 |

## 质检表(QA gate)怎么看

每份简历跑完,AI 会贴出一张表,每行一个检查项,三种结果:

- **✓ PASS** — 过了
- **⚠ WARN** — 提醒,不致命,但要让 AI 解释一句
- **✗ FAIL** — **没过,这份简历不算完成**,要让 AI 修到全绿

检查项大致分四类,看不懂英文也没关系,记住"哪类红了大概是什么问题":

| 类别 | 检查项(英文名) | 红了通常意味着 |
|------|------------------|----------------|
| **真实性**(最重要) | `numeric_subset` | 简历里出现了你"总简历"里没有的数字 → 八成是 AI 编了个数,让它删或改回真实数 |
| | `forbidden` | 出现了"还在学/经验有限"这类自我贬低词,或你不会的语言 |
| | `languages` / `contact` / `education` | 你的语言、电话邮箱、学历没正确出现(或被改写了) |
| **排版** | `pages` | 页数不对(要 1 页结果 2 页)→ 让它再删点内容 |
| | `overfull` / `bottom_ws` / `right_ws` | 版面溢出或留白太多 → 让它调 |
| **文件齐全** | `files` | 少了报告、关键词或决策记录文件 |
| **覆盖率** | `coverage` | 只是告诉你 JD 关键词覆盖了多少,不是错误 |

**你要做的很简单:看到任何 ✗,就跟 AI 说"这个 gate 没过,修好再给我"。** 真实性类(尤其 `numeric_subset`、`forbidden`)红了要多留个心——那往往是 AI 想夸大,被脚本拦下来了,正是这套工具的价值所在。

## 出问题了怎么办(照着说就行)

不用自己排查,把现象告诉 AI 让它处理。常见几种:

- **装环境卡住 / 命令报错** → 先回到"第 0 步",让帮你的人陪你把 Claude Code、XeLaTeX、poppler 装好。这步过不了,后面都跑不动。
- **AI 说"编译失败 / xelatex error"** → 直接说:"编译报错了,你看下日志,修好 .tex 再重编,直到出 PDF。"
- **某个 gate 一直 FAIL** → "这个 gate 还红着,告诉我具体哪里不满足,然后修好。"
- **它把课程项目说成上线系统 / 把级别往上抬** → "这是夸大,改回真实说法,并把缺的能力记进 GAP_LIST。"
- **它说"已编译/已通过"但没给你看命令输出** → "请真的运行 qa_one_job.py,把它打印的表格原样贴给我。"(规矩里要求它必须真跑,不能嘴上说。)

---

## 什么时候算"这份简历做完了"

满足下面全部才算(让 AI 对照确认):

1. PDF 编译成功,页数符合要求(一般 1 页);
2. `qa_one_job.py` **全部 PASS**(WARN 已解释),表格贴在报告里;
3. 报告里 `SUBSET_VIOLATIONS` 为空(没有无依据的内容);
4. JD 要而你没有的,都老实写进了 `GAP_LIST`(而不是编出来);
5. 这一份在 `PHONE_INTERVIEW_PREP/` 里有对应的电话小抄。

---

## 你的资料、你的隐私

- **全部在你自己电脑上跑**,你的简历和资料不会上传到别处,也不在这套工具的作者手里。
- `profile.yaml` 和那份"绝不夸大清单"是**你的**——只有你能定哪些是真、哪些不写。
- 想改信息(换电话、加一段新经历)随时改 `profile.yaml` 或让 AI 更新你的"总简历"即可,改完下次投递自动生效。

## 常见问题(FAQ)

**我可以用中文跟它说吗?** 可以,中英文都行,它都懂。工具里的文档是英文的,但那是给 AI 读的。

**要花钱吗?** Claude Code 需要一个 Claude 账号(订阅或 API)。XeLaTeX、poppler 是免费的。

**我还没有一份完整简历,能用吗?** 能。第 ③ 句(BOOTSTRAP)就是 AI 访谈你、帮你从零建起"总简历"。

**我做好的简历在哪?** 在 `JOB_APPLICATIONS/<公司名>/` 文件夹里,`.pdf` 就是成品。

**它会不会替我自动投递?** 不会。它只生成简历和面试小抄,投不投、投哪儿都是你自己来。
