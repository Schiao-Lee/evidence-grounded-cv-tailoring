The idea

AI is great at rewriting — and great at quietly overstating. This project treats truthfulness as an engineering constraint, not a hope. The CV is tailored down from a master profile you control, and a deterministic gate refuses to call anything "done" until every claim checks out. It makes honest tailoring faster, not looser.

Honest by design

Subset rule

The tailored CV may contain only facts already in your master profile. Nothing new is ever invented to match a job.

Gap rule

If the job needs something you don't have, it's logged as an explicit gap — surfaced honestly, never fabricated.

13 deterministic QA gates

A Python gate — not the model's word — verifies every number traces to source, contact / education / languages stay intact, no filler slips in, and layout holds.

Config-as-data

Zero personal facts in code. One profile.yaml drives every personalized check, so the same engine fits anyone.

Verification-first

Real XeLaTeX compile + PDF / coverage audits, run for real — no "compiled successfully" claims without the command output.

Your data stays local

The workflow runs on your machine. Nothing is uploaded. The examples shipped in the repo are entirely synthetic.

How it works

Three deliberately separated layers, so no single prompt owns everything:

Evidence master CV + project bank + evidence map — what may be said.
Decision JD parsing, content selection under the subset rule, gap-list generation — saved to a decision trace.
Presentation a LaTeX visual system (dense / balanced profiles).
Parse JD→ Select content→ Render (LaTeX)→ Keyword gate→ QA gate→ Subset audit→ Phone-prep
